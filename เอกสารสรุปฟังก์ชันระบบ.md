# เอกสารสรุปฟังก์ชันระบบ
## ระบบบริหารจัดการงานบุคคลและการอนุมัติเอกสารอิเล็กทรอนิกส์
**HR Operations, E-Approval & Onboarding System — บริษัท วิจิตรภัณฑ์ก่อสร้าง**

> เอกสารฉบับนี้สรุปว่า **ระบบที่ท่านจะได้รับทำอะไรได้บ้าง** ในเชิงการใช้งานจริง เพื่อให้ทีมงานและผู้บริหารของท่านพิจารณาขอบเขตงานร่วมกันก่อนเริ่มพัฒนา ส่วนรายละเอียดทางเทคนิคสำหรับฝ่ายไอที สรุปไว้ในหัวข้อท้ายเอกสาร

---
 
## 1. ภาพรวมระบบ

ระบบนี้เป็น **เว็บแอปพลิเคชัน** ใช้งานผ่านเบราว์เซอร์ รองรับทั้งคอมพิวเตอร์และมือถือ ไม่ต้องติดตั้งโปรแกรมที่เครื่องผู้ใช้ ทำงานบนฐานข้อมูลกลางตัวเดียว และมีระบบควบคุมสิทธิ์การเข้าถึงตามบทบาท (Role-based Access)

ระบบรวม **4 โมดูลหลัก** ที่เชื่อมโยงข้อมูลผู้ใช้ พนักงาน แผนก และหน่วยงานร่วมกัน:

| โมดูล | ชื่อ | หน้าที่โดยย่อ |
|:---:|---|---|
| 1 | บันทึกข้อความและอนุมัติอิเล็กทรอนิกส์ (E-Memo) | จัดทำหนังสือ/บันทึก ขออนุมัติ พร้อมลายเซ็นและอนุมัติออนไลน์ |
| 2 | รายงานและวิเคราะห์การปฏิบัติงาน | บันทึกงานรายวัน + OT ของพนักงานทุกหน่วยงาน พร้อม Dashboard |
| 3 | บริหารวงเงินสินเชื่อโครงการ | จัดเก็บและติดตามวงเงินสินเชื่อ การเบิกใช้ และวงเงินคงเหลือ |
| 4 | แนะแนวและติดตามพนักงานใหม่ 90 วัน | คลังข้อมูลและแผนติดตามพนักงานใหม่ให้ปรับตัวได้เร็ว |

### บทบาทผู้ใช้งาน (Roles)

| บทบาท | สิทธิ์การใช้งาน |
|---|---|
| **ผู้ดูแลระบบ (Admin)** | เข้าถึงและตั้งค่าได้ทั้งหมด จัดการผู้ใช้/หน่วยงาน/สิทธิ์ |
| **ผู้บริหาร (Executive)** | ดูภาพรวมทุกหน่วยงาน (รวมข้อมูลการเงิน) และอนุมัติรายการได้ |
| **เจ้าหน้าที่ HR** | จัดการข้อมูลเฉพาะหน่วยงานที่ตนรับผิดชอบ |

> ระบบรองรับโครงสร้างองค์กรแบบ **หลายหน่วยงาน** โดยแต่ละหน่วยงานมีหลายแผนก หลายตำแหน่ง และพนักงานรายบุคคล — จำนวนหน่วยงานปรับเพิ่ม/ลดได้เองในภายหลัง

## 2. โมดูลที่ 1 — บันทึกข้อความและอนุมัติอิเล็กทรอนิกส์ (E-Memo & E-Signature)

ระบบจัดทำเอกสารขออนุมัติ ส่งเข้าสู่ขั้นตอนการอนุมัติแบบลำดับขั้น และแจ้งเตือนผู้เกี่ยวข้องทางอีเมล พร้อมเก็บประวัติทุกขั้นตอน

### 2.1 การจัดทำเอกสาร
- **แบบฟอร์มขออนุมัติสำเร็จรูป** เช่น ขออนุมัติทั่วไป / ขออนุมัติ OT / ขออนุมัติจัดซื้อ–เบิกจ่าย
- **สร้างหนังสือ/บันทึกแบบทางการอัตโนมัติ** พร้อมหัวจดหมาย (Letterhead) ตามหน่วยงาน/โครงการ — ผู้ใช้พิมพ์เพียงเนื้อความ ผู้รับ (เรียน) และเรื่อง ระบบประกอบเป็นหนังสือฉบับเต็มให้
- **ออกเลขที่เอกสารอัตโนมัติ** ในรูปแบบมาตรฐานองค์กร (เช่น `อักษรย่อ / แผนก / รหัสเอกสาร / เลขรันนิ่ง`) โดยเลขรันนิ่ง**แยกตามโครงการ** และระบบหาเลขล่าสุดมาต่อให้อัตโนมัติ
- **แนบไฟล์ประกอบ** ได้ และจัดเก็บเข้าที่จัดเก็บของโครงการอย่างเป็นระเบียบ

### 2.2 ขั้นตอนการอนุมัติ (Workflow)
- **เส้นทางอนุมัติแบบลำดับขั้น**: ผู้ขอ → ผู้อนุมัติลำดับที่ 1 → ลำดับที่ 2 → … (กำหนดผู้อนุมัติตามประเภทเอกสารได้)
- **แจ้งเตือนทางอีเมลทุกขั้นตอน** พร้อมปุ่ม **อนุมัติ / ไม่อนุมัติ / ส่งกลับแก้ไข** ที่กดดำเนินการได้จากอีเมล
- **ลายเซ็นอิเล็กทรอนิกส์**: รูปลายเซ็นและชื่อผู้อนุมัติ พร้อมตราเวลา (Timestamp)
- **ติดตามสถานะแบบเรียลไทม์** ว่าเอกสารอยู่ที่ขั้นใด ใครยังไม่ดำเนินการ

### 2.3 การจัดเก็บและตรวจสอบ
- **ประวัติการดำเนินการทุกขั้น (Audit Trail)** ตรวจสอบย้อนหลังได้ว่าใครทำอะไร เมื่อใด
- **จัดเก็บเอกสารที่อนุมัติแล้วเป็นไฟล์ PDF** ค้นหาย้อนหลังได้
- **ทะเบียนคุมเอกสาร** รวมศูนย์ บันทึกทุกเอกสาร (เลขที่ วันที่ ผู้ส่ง เรื่อง ประเภท โครงการ ลิงก์ไฟล์) ค้นหา/กรองตามโครงการ/ช่วงเวลาได้
- **บันทึกเอกสารจากอีเมลอัตโนมัติ**: ระบบช่วยดึงเอกสารที่ส่งเข้ามาทางอีเมลมาลงทะเบียนให้ ลดงานคีย์ข้อมูลซ้ำ
- **เปิดดูไฟล์เอกสารในระบบ** ได้ทันที (PDF / รูปภาพ ฯลฯ)
- รองรับ **วันที่แบบไทย (พ.ศ.)** ในเอกสาร

## 3. โมดูลที่ 2 — รายงานและวิเคราะห์การปฏิบัติงาน

ระบบบันทึกการปฏิบัติงานรายวันของพนักงานทุกคนในแต่ละหน่วยงาน พร้อมข้อมูล OT และสรุปภาพรวมเป็น Dashboard

### 3.1 รูปแบบการบันทึก
- **เจ้าหน้าที่ HR บันทึกแทนพนักงานทั้งหน่วยงาน** ในหน้าจอเดียว (ไม่ต้องให้พนักงานกรอกเอง)
- รองรับพนักงาน **2 ประเภท** ที่มีข้อมูลต่างกัน:
  - **สายปฏิบัติการ** — บันทึกทีม/ลักษณะงาน และ **ชั่วโมง OT** รายวัน
  - **สายสนับสนุน** — บันทึกรายละเอียดงานรายวัน (ไดอารี่งาน) และหมายเหตุ
- ข้อมูล OT ครบถ้วน: ชั่วโมง OT, อัตรา, จำนวนเงิน, เหตุผล/รายละเอียด

### 3.2 หน้าจอกรอกข้อมูล (ออกแบบให้กรอกเร็ว)
- **ตารางรายเดือน (พนักงาน × วัน)** กรอกข้อมูลในช่องได้ทันที
- **มุมมองรายสัปดาห์** เลื่อนดูทีละสัปดาห์ พร้อมไฮไลต์วันนี้และวันหยุด
- **บันทึกอัตโนมัติ** หลังพิมพ์ไม่กี่วินาที พร้อมแสดงสถานะ (กำลังบันทึก / บันทึกแล้ว)
- **ตัวช่วยเลือกประเภทงาน** ค้นหาได้ จัดกลุ่มตามหมวด จากทะเบียนประเภทงานกลาง
- **ล็อกการแก้ไขย้อนหลัง**: ข้อมูลที่เกินกำหนด (เช่น 3 วัน) จะล็อกอัตโนมัติเพื่อความถูกต้อง โดยผู้ดูแลระบบสามารถปลดล็อกแก้ไขย้อนหลังได้

### 3.3 การติดตามความครบถ้วน (กันข้อมูลตกหล่น)
- **มุมมองภาพรวมความครบถ้วน (Coverage)** แสดงเป็นตารางสีว่าใครบันทึก/ยังไม่บันทึกในวันใด (บันทึกแล้ว / ขาด / ลา / วันหยุด / ยังไม่ถึงกำหนด)
- **สรุปเปอร์เซ็นต์ความครบถ้วนรายวัน** (เขียว ≥80% / เหลือง 40–79% / แดง <40%)
- คลิกที่ช่องเพื่อกระโดดไปแก้ไขวันนั้นได้ทันที

### 3.4 Dashboard และรายงาน
- **Dashboard รายหน่วยงาน** แสดงสรุปและความคืบหน้าของแต่ละหน่วยงาน
- **สรุปและเปรียบเทียบหลายหน่วยงานรายเดือน** พร้อมกราฟแนวโน้ม
- **ส่งออกรายงาน** เป็น Excel และ PDF ตามหน่วยงาน / แผนก / ช่วงเวลา

### 3.5 การจัดการข้อมูลพื้นฐาน
- จัดการทะเบียน **หน่วยงาน / แผนก / ตำแหน่ง / พนักงาน**
- จัดการ **ผู้ใช้และสิทธิ์** (ผู้ดูแลระบบกำหนดอีเมล หน่วยงานที่รับผิดชอบ และบทบาท)

## 4. โมดูลที่ 3 — บริหารวงเงินสินเชื่อโครงการ (Credit Facility Management)

ระบบจัดเก็บและบริหารข้อมูลวงเงินสินเชื่อของแต่ละโครงการไว้ในที่เดียว ทดแทนการเก็บข้อมูลกระจัดกระจายในไฟล์ Excel พร้อมควบคุมสิทธิ์การเข้าถึงข้อมูลการเงินที่เป็นความลับ

### 4.1 ทะเบียนวงเงินสินเชื่อ
- บันทึกข้อมูล **โครงการ** และ **วงเงินสินเชื่อรายโครงการ** โดยรองรับ **หลายวงเงิน / หลายสถาบันการเงินต่อหนึ่งโครงการ**
- รายละเอียดต่อวงเงิน: สถาบันการเงิน, ประเภทวงเงิน (เช่น หนังสือค้ำประกัน L/G, เงินกู้ T/L, ตั๋วแลกเงิน B/E, ตั๋วสัญญาใช้เงิน P/N), จำนวนวงเงินที่อนุมัติ, อัตราดอกเบี้ย/ค่าธรรมเนียม, วันที่อนุมัติ และวันครบกำหนด
- **คำนวณวงเงินคงเหลืออัตโนมัติ** (วงเงินที่อนุมัติ − วงเงินที่ใช้ไป)
- **ปรับวงเงินได้** เมื่อสถาบันการเงินเปลี่ยนแปลงวงเงิน

### 4.2 การติดตามการเบิกใช้และคำขอ
- บันทึกรายการ **เบิกใช้ (Drawdown)** และ **ชำระคืน** แต่ละครั้ง
- **กฎการนับวงเงินที่ใช้ไป** ชัดเจน: รายการที่ **อนุมัติแล้ว** จึงนับเป็นยอดใช้วงเงิน, รายการ **คำขอใหม่/รออนุมัติ** บันทึกไว้แต่ยังไม่กินวงเงิน, รายการ **ชำระแล้ว** จะคืนวงเงินให้อัตโนมัติ
- **ระบบคำขอ–อนุมัติ**: ยื่นคำขอใช้วงเงิน → ผู้บริหารอนุมัติ/ไม่อนุมัติ — เมื่ออนุมัติแล้วระบบสร้างรายการใช้วงเงินจริงให้อัตโนมัติ (ไม่นับซ้ำ)

### 4.3 การติดตามและแจ้งเตือน
- **จัดกลุ่มตามวันครบกำหนด**: เกินกำหนด / ครบเดือนนี้ / เดือนหน้า / ภายหลัง เพื่อติดตามวงเงินใกล้ครบกำหนด
- **คำนวณดอกเบี้ยของรายการที่เกินกำหนด** ให้โดยอัตโนมัติ

### 4.4 แผนกระแสเงินสดรายเดือน (Cash Plan)
- วางแผนรายงวด/รายเดือนต่อโครงการ: รายรับ, รายการที่จะชำระในงวดนั้น, ตั๋วสัญญาใช้เงินใหม่, รายการหัก และรายละเอียดรายรับ
- ช่วยให้เห็นภาพการเงินของโครงการล่วงหน้า

### 4.5 ความปลอดภัยและการตรวจสอบ
- **ประวัติการแก้ไขข้อมูลแบบละเอียด (Audit Trail)** บันทึกผู้แก้ไข เวลา และค่าก่อน–หลังการแก้ไขทุกครั้ง
- **ควบคุมสิทธิ์การเข้าถึง** — เฉพาะผู้บริหาร/ผู้ดูแลระบบเท่านั้นที่เห็นข้อมูลการเงินและอนุมัติได้
- **ค้นหาและกรอง** ตามโครงการ / ประเภทวงเงิน / สถานะ / ช่วงครบกำหนด
- **ส่งออกข้อมูลเป็น Excel** ตามเงื่อนไขที่เลือกบนหน้าจอ

## 5. โมดูลที่ 4 — แนะแนวและติดตามพนักงานใหม่ 90 วัน

ศูนย์รวมข้อมูลและแผนงานสำหรับพนักงานใหม่ ช่วยให้ปรับตัวได้เร็วและเป็นมาตรฐานเดียวกันทุกหน่วยงาน

### 5.1 คลังข้อมูลพนักงานใหม่
- รวบรวม **นโยบายบริษัท สวัสดิการ คู่มือการทำงาน เอกสารที่ต้องลงนาม และสื่อแนะนำองค์กร** ไว้ในที่เดียว
- พนักงานใหม่เข้าถึงข้อมูลที่จำเป็นได้ด้วยตนเอง

### 5.2 แผนแนะแนวและติดตามผล
- **แผนแนะแนว 30–60–90 วัน** — รายการสิ่งที่ต้องทำในแต่ละช่วง
- **ติดตามความคืบหน้า** ว่าพนักงานใหม่ทำครบตามแผนหรือยัง
- **แบบประเมินทดลองงาน** สำหรับหัวหน้างาน/HR

> หมายเหตุ: โมดูลนี้พัฒนาตามขอบเขตในข้อเสนอโครงการ และเราจะปรับรายละเอียดให้ตรงกับกระบวนการรับพนักงานใหม่จริงของบริษัทท่านในช่วงเริ่มพัฒนา

## 6. คุณสมบัติที่ใช้ร่วมกันทุกโมดูล

- **ใช้งานผ่านเบราว์เซอร์** ทั้งคอมพิวเตอร์และมือถือ (Responsive)
- **เข้าสู่ระบบด้วยอีเมลและรหัสผ่าน** จัดการสิทธิ์ตามบทบาท
- **ฐานข้อมูลกลางตัวเดียว** ข้อมูลผู้ใช้/พนักงาน/หน่วยงานเชื่อมโยงกันทุกโมดูล
- **รองรับภาษาไทยและปีพุทธศักราช (พ.ศ.)** เต็มรูปแบบ
- **บันทึกประวัติการใช้งานและการแก้ไข** เพื่อความโปร่งใสและตรวจสอบย้อนหลัง
- **เชื่อมต่อกับอีเมลองค์กร** สำหรับการแจ้งเตือนและอนุมัติ

## 7. สรุปเชิงเทคนิค (โดยย่อ)

หัวข้อนี้จัดทำไว้สำหรับฝ่ายไอทีของท่านโดยเฉพาะ เพื่อให้เห็นภาพแนวทางการพัฒนา ทั้งนี้รายละเอียดเฉพาะเจาะจง เรายินดีหารือร่วมกับฝ่ายไอทีของท่านในช่วงเริ่มโครงการ เพื่อให้สอดคล้องกับโครงสร้างพื้นฐานและนโยบายความปลอดภัยที่ท่านใช้อยู่

| องค์ประกอบ | แนวทางที่ใช้ |
|---|---|
| **ส่วนผู้ใช้งาน (Front-end)** | เว็บแอปแบบ Responsive พัฒนาด้วย React (Vite) และ Tailwind CSS |
| **ส่วนประมวลผล (Back-end)** | บริการ API กลางด้วย Node.js จัดการสิทธิ์ผู้ใช้และตรรกะทางธุรกิจทั้งหมด |
| **ฐานข้อมูล** | ฐานข้อมูล MongoDB บน MongoDB Atlas พร้อมการสำรองข้อมูล |
| **ระบบยืนยันตัวตน** | จัดการในระบบเอง (อีเมล + รหัสผ่านเข้ารหัส bcrypt) ออกโทเคน JWT ควบคุมสิทธิ์ตามบทบาท |
| **ที่จัดเก็บไฟล์** | GridFS ในฐานข้อมูล MongoDB สำหรับไฟล์แนบและเอกสาร PDF (เข้าถึงผ่าน API ที่ต้องยืนยันตัวตน) |
| **งานอัตโนมัติตามเวลา** | งานที่ทำงานเป็นรอบ เช่น ดึงเอกสารจากอีเมล สรุปรายเดือน และคำนวณดอกเบี้ย/วันครบกำหนด |
| **การแจ้งเตือน/อนุมัติทางอีเมล** | เชื่อมต่อระบบอีเมลองค์กร (Google Workspace / Microsoft 365) |
| **การติดตั้ง (Hosting)** | ให้บริการบนคลาวด์ Railway (รองรับการขยายระบบและใบรับรองความปลอดภัยในตัว) |
| **ความปลอดภัย** | เข้ารหัสการเชื่อมต่อ (HTTPS), สิทธิ์ตามบทบาท, บันทึกการใช้งาน (Audit Log) |

### หลักการออกแบบ
- **API กลางเป็นศูนย์รวมตรรกะทางธุรกิจ** ทุกการคำนวณสำคัญ (เช่น การนับวงเงินใช้ไป, เลขรันนิ่งเอกสาร, เส้นทางอนุมัติ) ทำที่ฝั่งเซิร์ฟเวอร์เพื่อความถูกต้องและปลอดภัย
- **ควบคุมสิทธิ์หลายชั้น** ทั้งที่ระดับแอปพลิเคชันและระดับฐานข้อมูล
- **สิทธิ์ในซอร์สโค้ดเป็นของลูกค้า** ตามเงื่อนไขในข้อเสนอโครงการ

## 8. ขอบเขตและการตรวจรับ

- ขอบเขตงานอ้างอิงตามที่ระบุในเอกสารนี้และข้อเสนอโครงการ หากมีการเปลี่ยนแปลงหรือเพิ่มเติม เราจะประเมินเวลาและค่าใช้จ่ายเพิ่มเติมให้ท่านพิจารณาก่อนทุกครั้ง
- เราส่งมอบระบบแบบแบ่งระยะ (4 ระยะตามโมดูล) เพื่อให้ท่านเห็นผลลัพธ์เป็นช่วง ๆ และให้ข้อเสนอแนะได้ระหว่างทาง
- รับประกันการแก้ไขข้อบกพร่อง 90 วันหลังส่งมอบ และมีแพ็กเกจดูแลบำรุงรักษารายปี
